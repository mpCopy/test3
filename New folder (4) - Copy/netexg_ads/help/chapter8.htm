Drills

ODB and IPC-2581 do not store via information directly. Instead they store the drill data that will be used to manufacture the board. The PCB importer reads the drill information and converts this into the required entities the dielectric layers that act to interconnect the conductor layers.

The user generally does not need to make any adjustments to the drill conversion except for a single case - to instruct the program to identify back drills.

The drill dialog is accessed from the main dialog by clicking on the "Drill ..." button.

Fields and Entries

Gerber File - one or more drill files which have been converted into Gerber format. This is done automatically when the drill data in the ODB or IPC file is detected and processed. No user intervention is needed here.

Top Layer - the top layer (i.e. layer 1) where the drill begins. The number is the "stackup" layer. "1" indicates the top conductor layer.

Bottom Layer - the bottom layer (i.e. layer 5) where the drill ends. The number is the "stackup" layer. Drills are assumed to go from top to bottom except for the special case of back drills.

Boards with buried or blind vias generally will have more than one drill file associated with them.

Drill Settings

    Check for Back Drills - if checked, the program will check each drill file against all the others in order to determine if the coordinates line up. If it detects such a condition it will assume that one of the files is a back drill and will use it to remove the via that would have otherwise been produced by the front drill.

    For a detailed explanation of backdrilling see this page: Back Drilling to Reduce Via Parasitics

      Snap - when checking for back drills a small snap radius can be defined so that tiny variations in the X,Y location will be absorbed. Such a snap distance should be set to be 10X smaller than the smallest drill diameter.


      Arc Res - drills are circular but are broken into segments. The arc res determines the number of segments based on how many degrees seach segment subtends. If set = 0 then it has no effect.



      Chord Error - chord error is an alternate approach to determining how many segments are used to approximate the drill. If set = 0 then it has no effect




      No of Sides - the user can specify directly the number of sides of the polygon use to represent the drill. Must be a value of 4 or greater.


    Drop Vias in Output - if checked the vias will not be generated in the output. This option is generally not used if the ADFI is intended for simulation but might be useful if one wants to make a 3D display of the board while minimizing the number of 3D entities.


Mount Hole Settings

    Detect Mount Holes - This is used to differentiate drills used to produce vias from drills used to produce mounting holes. Any drill whose diameter exceeds the size parameter will be treated as a non plated mounting hole. Generally speaking this has little relevance to simulation.
