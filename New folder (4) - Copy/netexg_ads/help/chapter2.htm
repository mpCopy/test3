About ODB++ and IPC2581

ODB++ and IPC2581 are manufacturing data formats. These formats are normally exported from a completed board layout and sent to the PCB fabricator. The contain the geometry for each conductor layer and all of the drill information for creating vias. They should also contain component and net information (though this is not 100% mandatory)

ODB++

ODB++ was created by Valor and is a quasi proprietary format. However it has been used in the PCB industry for over 15 years and the great majority of PCB layout tools can produce this format.

The actual database is not a single file -- rather it is organized into a strict hierarchy of directories and files.


For purposes of exchanging ODB++ files, the Valor standard is to use the Unix tar commmand to roll up all the directories and files into a tar file and then to use the gzip compression utility to compress the tar file. When you see a ODB++ file with the extension .tgz you know that it is a tar-gzipped file.

The PCB Importer can read either the .tgz form (in which case it automatically unpacks the data into a temporary directory) or it can read the directory hierarchy when the user points to the top of the hierarchy.

Many Windows based PCB layout tools do not produce the .tgz format. Instead they rely on the zip utility to combine and compress the directory hierarchy. Because there are variations in zip files, The PCB Importer does not read the ODB++ file directly from a zip archive. Instead, the user must first unzip the data and then point NETEX-ADS to the top of the hierarchy.

IPC2581

IPC2581 is a newer manufacturing output. Unlike ODB++, this is a non proprietary standard created by the IPC. However very few programs produce the format at this time.

IPC2581 delivers essentially the same information as ODB++ but contained in a single XML file. One advantage of IPC2581 over ODB++ is that it generally contains usable information about each conductor and dielectric layer's thickness and material which is not present in ODB++.
