*******************************************************
----------- Generation of Momentum Module -------------
*******************************************************


The Momentum Module generation can be said as a 2 step process;

1.	Generation of the Substrate File(*.ltd file).
		The Substrate file requires 3 input files namely, Procfile, P2lvsfile and Techfile. The Perl Script 'MomICSub.pl' takes these 3 inputs and
	generates the Substrate file.
	
2.	Generation of the complete Momentum Module.
		The Momentum Module would be generated by the Perl Script 'MomICMod.pl'. Some files that need to be copied into the Momentum Module are placed
	in a folder named 'source' inside the 'HPEESOF_DIR/cdsubed/emmodule/'. The generated Substrate file will also be copied into the Momentum module.


There are 2 ways for generating the Momentum module;

1.	By invoking the 'Virtuoso Substrate Editor'
		The Virtuoso Substrate Editor makes use of a User Interface(UI) to take in the inputs from the user. The UI then calls the Perl scripts for the generation
	of Substrate file and the Momentum module.
	
2. 	By using 'Command Line'
		The user has to enter all the arguments required on the command line, with each argument seperated by a space. The procedure for running the Perl
	scripts on the command line is explained in detail below.
	

--------------------------------------------------------------------
	   Command Line approach for generating Momentum Module	
--------------------------------------------------------------------

1.	Set the HPEESOF_DIR environment variable
2.	Run a Perl script to generate LTD substrate file(s)

    From Cadence tech files: run $HPEESOF_DIR/cdsubed/emmodule/input/MomICSub.pl

	    $HPEESOF_DIR/cdsubed/emmodule/input/MomICSub.pl <GroundPlaneFlag(integer)> <Substrate_Name(string)> <Substrate_Resistivity(integer)> <Substrate_Permittivity(integer)> 
	    <Substrate_Height(integer)> <Procfile_path(string)> <P2lvsfile_path(string)> <Techfile_path(string)> <SmartLayerMergeFlag(string) - "yes"/"no"> 
	    <SubstrateFileName(string)> <SubstrateFile_path(string)>

    From iRCX files, run $HPEESOF_DIR/cdsubed/emmodule/input/MomICSub.pl

        Usage: MomICSub_iRCX.pl [<options>...] <techFilePath> <ircxFilePath> [<ircxFilePath>...]

        <techFilePath>                  Cadence technology file name, including the path
        <ircxFilePath>                  iRCX file name, including the path

        <Options>
           --bsm                        Add back side metalization
           --mVias                      Use derived layer for automatic via array simplification
           --mDiels <1|2>               Merge dielctric layers, 1 = intra-metal, 2 = inter-metal
           --lyrmp <layermapFilePath>   Override layer mapping
           --ltdSuffix <ltdSuffix>      Set suffix for substrate output file (".ltd")
           --dltfSuffix <dltfSuffix>    Set suffix for derived layer tech output file ("_em.tf")
           --dlNrOffset <offset>        Set offset of derived layer numbers (1000)
           --oDir <outputDir>           Set output directory

        This script generates LTD substrate output files, named after the iRCX files with <ltdSuffix> suffix. 
        Optionally, a technology file with derived layer definitions, named after the Cadence technology file with <dltfSuffix> suffix.

3. Run the Perl script, $HPEESOF_DIR/cdsubed/emmodule/MomICMod.pl, to generate the Momentum Module

        Usage: MomICMod.pl [<options>...] <moduleName> <techFilePath> <ltdFilePath> [<ltdFilePath>...]

        <Options>
            --dltf <dlTechFilePath>      Cadence tech file with derived layers
            --lyrmp <layermapFilePath>   Layer mapping file
            --oDir <outputDir>           Set output directory
		
	NOTE : All strings should be enclosed within double-quotes(") and the paths should have forward-slashes only(/).
